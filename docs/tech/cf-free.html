              # 2. 完整思路

              注意事项

              以下通过Pages部署的方式（第5-9步）好像已经失效，建议使用 Workers 部署 (opens new window)

              1. 注册免费域名（教程），例如：jack.dpdns.org
              2. 注册 Cloudflare 账号
              3. 在 Cloudflare 中托管域名（教程）
              4. 新建 Worker KV
              5. 新建 Pages，上传压缩包 (opens new window)并部署
              6. 添加环境变量 ADMIN，并设置密码，例如：123
              7. 绑定子域名，例如：cf.jack.dpdns.org
              8. 绑定刚才的 KV
              9. 重新上传压缩包 (opens new window)，重新部署 Pages
              10. 访问控制台：https://cf.jack.dpdns.org/admin，输入密码：123
              11. 复制订阅链接
              12. 根据自己的系统，下载并安装客户端，例如：V2RayN
              13. 打开客户端，按 Ctrl + V，导入订阅链接
              14. 选择 订阅分组 > 更新全部订阅（不通过代理），拿到优选节点
              15. Ctrl + A 全选节点，Ctrl + R，测试所有节点真实速度
              16. 随便选择一个节点，按 Enter 键
              17. 在最下方选择 自动配置系统代理，开始翻墙
              18. 打开 lite.ipcheck.ing (opens new window)，查看自己 IP 所在的地区
              19. 在最下方选择 清除系统代理，结束翻墙
              20. 根据 设置 Workers/Pages 可用请求数统计 (opens new window) 在Admin页面，显示每日请求额度使用情况
